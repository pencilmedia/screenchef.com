AGILE 8 VS. AGILE 9

1. What key productivity features did you GAIN coming to 9.0 from 8.5? Nothing.

During our rollout to A9, we also went through a new PN system, Oracle 11i, we had too many changes going on at once. We now require users to add lots of attributes when creating parts. Old part numbers need to be updated to this new process. 

2. What key productivity features did you LOSE coming to 9.0 from 8.5?

  • Takes a lot more clicks. (refresh/save)
  • Lot more convoluted access to information. (lack of inline table editing)
  • Takes longer to use. (performance)
  • Working with attachments
  • Save As
  • Building part number descriptions (with process extension)

3. Have you changed how you use Agile in 9 compared to 8? How?

  • We now have the new (catalog) PN process.
  • Using the star key was new.
  • The file system is different.
  • Lots of pop ups (no inline table editing)
