HAND DATA ENTRY CASES

1. Describe the process you follow to enter PARTS by hand. What works well in Agile? What about Agile makes this difficult? Need to enter free text in the part description before running the Px (which generates the item description based on P3 values). [seems that the Px appends to whatever is in the description field]

Logitech introduced catalog values, but the data is too restrictive.  There are not enough options.

2. Describe the process you follow to enter / modify BOMs by hand. What works well in Agile? What about Agile makes this difficult?

  • Love part number drag-n-drop.
  • Have to hit refresh before adding each new part in both Item.BOM and ECO.AI.RL BOM tables.
  • Same with Copy and Paste.

3. Describe your change management process? How are BOMs, AMLs, Drawings changed during the ECO process? What works well in Agile? What about Agile makes this difficult?

  • ECO AI editing requires a form be completed (fixed in 92). Should be a table.  Editing AI (form Edit).  Can't copy/paste into BOM edit.
  • Adding parts to a BOM.  
  • Clicking Refresh all the time to avoid the "someone is working on this object" error.
