HAND DATA ENTRY CASES

1. Describe the process you follow to enter PARTS by hand. What works well in Agile? What about Agile makes this difficult? Engineer creates part. In past engineer created temporary part. Now with Ora Mfg 11i, e/one creates parts directly in Agile now. Based on PN, a minimum # of req'd fields are attributed on P3; these are called catalog fields. Px used to auto-generate a part described based on catalog values.

Keyboard / Language versions ... do a save-as. Change different fields. Some people create parts and import.

2. Describe the process you follow to enter / modify BOMs by hand. What works well in Agile? What about Agile makes this difficult? Some do import. Some type=in known number or run search then drag and drop to BOM. The 9.0 Edit Form (now replaced by table in 92) is not useful. 

3. Describe your change management process? How are BOMs, AMLs, Drawings changed during the ECO process? What works well in Agile? What about Agile makes this difficult? If part is at pre-release, no CCB needd. If part is released (active status), goes through CCB and is released to Ora Mfg and scheduled. Once Ora Mfg's auto-implement manager runs, Agile is programatically updated and the agile WF moves to Implemented.

AML updated via MCO. In the past the revision and status were tied together.

Engineers cannot live without Agile!  Engineers are used to the way it works (considered a critical application), and they get upset if things change.

2. What do you find better in the Web client vs. the Java Client? Casual user. Quick search cases. Can't copy disabled fields in Java, need to go to web.

4. What do you find inefficient in the Java Client? What workarounds have you come up with? Can't copy and paste consistently across all windows / fields. If a field is disabled but visible it cannot be select and, thus, copied to the clipboard.

AGILE 8 VS. AGILE 9

1. What key productivity features did you GAIN coming to 9.0 from 8.5?

[EXTREME LAUGHTER]. That's a bad question. We went backwards. Went ot 9 for unlimited P2/P3 fields used now for catalogs (auto-description generator / see above) and for sites which didn't work for us.

2. What key productivity features did you LOSE coming to 9.0 from 8.5?

  • Performance
  • Usability - BOM/AI editing (now a form vs. inline table).  The popup threw people off.
  • Really liked 8.5. Hate 9.0.
  • Refresh is horrible in 9. - tell our users to "always refresh" as a mandatory action otherwise they experience "someone is working on this object" message along with data loss. Sometimes this message appears even when nobody else is working on the object. (Primarily on Java client)

3. Have you changed how you use Agile in 9 compared to 8? How?

  • Have to refresh often.
  • Tell users to click slower to avoid someone working on object message.
  • Takes more time for people to get their jobs done.
