This is a follow-up to add something to yesterday's survey.

As I stated, in the web client clicking to traverse through the web client is time consuming, especially remotely.  For instance, I am reviewing an MCO which is releasing about 10 components.  To release a component one of the items on my checklist is to make sure the Manufacturer's part has a specification attached.  It is not my job as a Change Analyst to insure the attachment is correct.
 
So, to do this I start at the affected items tab of the MCO.
  • Go to first AI
  • Click on the Manufacturer's tab
  • Click the first MPN
  • Check to make sure there is at least one attachment (I just look for the paperclip on the tab now)
  • Go back
  • Do as many times as appropriate for each MPN
  • Go back to AI tab
  • Do the same for each AI

This is quite tedious.

A couple suggestions:

  • Clicking on a MPN from an Agile PN brings you by default to the MPN General Info tab. It would be nice if the could go directly to any tab.  And it would be nice if the link to the tabs would give an indication of if something was in the tab. So for example, if at the Manufacturer's tab for an Agile PN, it would be nice if there was a column for attachments.  Clicking a hollow paperclip icon would bring directly to the MPN attachment tab.  A yellow paperclip would do the same but would let you know there is currently an attachment.

  • When I go from the MPN back to the Agile PN (either by cookie crumb or back button) it brings me back to the last released version.  However, I was previously in the pending version.  So, that's one more annoying click.
