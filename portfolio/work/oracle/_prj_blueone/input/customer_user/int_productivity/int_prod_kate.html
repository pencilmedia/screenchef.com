2. What are the most tedious / time consuming tasks you do IN AGILE? What should Agile change in the product to fix this?
Part number creation - process issue that engineers use a Part Request, not direct number creation. 
When processing an NPR, save as new component (change Subclass).  Not all P3 data copies over.  Open 2 objects side by side.  Copy / paste data from one window (attribute data).

HAND DATA ENTRY CASES

1. Describe the process you follow to enter PARTS by hand. What works well in Agile? What about Agile makes this difficult?

Direct entry on temp object - NPR.  Save As from NPR, copy missing data, copy Datasheet.  Save As does not bring over all P3 data. 

Introductory rev is bad and sometimes leaves attachments basically stuck on Intro rev and not very visible. 

Keep 2 windows open within JC to compare object data and copy missing.


2. Describe the process you follow to enter / modify BOMs by hand. What works well in Agile? What about Agile makes this difficult?

Adding rows puts them at bottom of page - hard to fish around for the new row.  Changing a redline twice will make it disappear on release.  Config Mfr prints ECO - truncates redlines on Print.
